### 分析与讨论
#### 1. 文本标签`<strong>`和`<b>`有什么区别？什么时候应该使用哪个？
（1）语义区别：<strong>是语义化标签，表示 “重要性强调”，浏览器默认加粗显示，同时向搜索引擎和辅助技术传递 “该内容重要” 的信息；<b>是表现型标签，仅表示 “视觉加粗”，无语义含义。
（2）使用场景：需要强调内容的重要性（如警告信息、核心观点）时使用<strong>；仅需视觉上加粗文本（如标题中的某些词汇）且无重要性强调时，可使用<b>。推荐优先使用<strong>，符合 HTML 语义化原则。

#### 2. 为什么在插入图像时必须提供`alt`属性？
（1）替代文本功能：当图像因路径错误、网络问题或文件损坏无法加载时，alt属性的文本会显示在图像位置，让用户了解图像内容。
（2）无障碍访问：屏幕阅读器等辅助技术会读取alt属性的文本，帮助视觉障碍用户理解图像信息，符合 Web 无障碍标准。
（3）搜索引擎优化：搜索引擎无法直接识别图像内容，alt属性的文本可帮助搜索引擎理解图像主题，提升页面 SEO 效果。

#### 3. 链接的`target`属性有哪些取值？各有什么作用？
（1）_self：默认值，在当前窗口 / 标签页打开链接，覆盖当前页面。
（2）_blank：在新的空白窗口 / 标签页打开链接，不影响当前页面。
（3）_parent：在父框架集中打开链接，适用于框架页面（frameset）。
（4）_top：在整个窗口中打开链接，忽略所有框架结构。
（5）自定义名称：在指定名称的窗口 / 标签页中打开链接，若该窗口不存在则新建。
（6）常用场景：跳转至外部网站时使用_blank，避免用户离开当前网站；跳转至站内页面时使用_self。

#### 4. 表格的`thead`和`tbody`标签有什么作用？
（1）结构语义化：<thead>定义表格头部（表头行），<tbody>定义表格主体（数据行），明确表格的结构划分，使代码更具可读性和维护性。
（2）样式控制：可通过 CSS 分别对表头和表体设置不同样式（如表头背景色、表体行 hover 效果），提升表格视觉区分度。
（3）动态操作：JavaScript 可单独操作<tbody>中的数据行（如添加、删除、排序），不影响表头结构，简化开发。
（4）打印优化：打印网页时，<thead>可设置为每页重复显示，方便长表格阅读。

#### 5. 表单的`method`属性有哪些取值？各有什么特点？
（1）get：默认取值，表单数据通过 URL 传递，数据会显示在地址栏中。
特点：传递数据量有限（通常不超过 2KB）；数据可见，安全性低；可缓存，支持书签收藏；适用于查询、搜索等无敏感数据的场景。
（2）post：表单数据通过 HTTP 请求体传递，数据不显示在地址栏中。
特点：传递数据量无明确限制（取决于服务器配置）；数据不可见，安全性较高；不缓存，不支持书签；适用于提交敏感数据（如用户名、密码）、上传文件等场景。

### 实验总结
通过本次实验，我们学习了HTML中常用的各种标签，包括文本标签、列表标签、图像标签、链接标签、表格标签和表单标签。通过实际操作，我们掌握了如何使用这些标签来创建具有不同内容和功能的网页元素。我们了解了各种标签的语义和适用场景，以及它们的基本属性和用法。

### 思考与拓展
#### 1. 为什么HTML5强调语义化标签？语义化标签有什么优点？
（1）HTML5 强调语义化标签的核心原因是解决传统 HTML 中 “用表现型标签描述结构” 的问题（如用<div class="header">替代<header>）。
（2）优点：① 提升代码可读性和维护性，使开发者快速理解页面结构；② 帮助搜索引擎抓取页面核心内容，提升 SEO 效果；③ 支持辅助技术（如屏幕阅读器），保障无障碍访问；④ 减少不必要的 class 和 id 命名，简化代码。

#### 2. 在实际开发中，如何选择合适的标签来展示内容？
（1）优先遵循 “语义优先” 原则：根据内容的语义选择标签，而非仅根据视觉效果（如强调重要内容用<strong>而非<b>）。
（2）匹配内容场景：如排序内容用<ol>，无序内容用<ul>，术语解释用<dl>；表格仅用于展示结构化数据，不用于页面布局。
（3）考虑兼容性：对于老旧浏览器不支持的 HTML5 语义化标签（如<header>、<nav>），可通过 CSS 设置display: block兼容。
（4）简化结构：避免嵌套过深，选择最简洁的标签实现需求（如用<p>包裹段落，而非多个<br>）。

#### 3. 如何优化表单的用户体验？
（1）明确标签关联：使用<label for="id">与表单元素关联，点击标签可聚焦对应元素，提升操作便捷性。
（2）提供即时反馈：通过 HTML5 内置校验（required、pattern）或 JavaScript 实现实时输入校验，显示错误提示。
（3）合理分组：使用<fieldset>和<legend>对表单元素分组（如个人信息、联系方式），提升可读性。
（4）优化输入体验：为文本框添加placeholder提示，下拉框设置默认选项，密码框提供 “显示 / 隐藏” 切换功能。
（4）按钮设计：提交和重置按钮样式区分明显，按钮位置靠近表单底部，符合用户操作习惯。

#### 4. 表格在现代网页设计中的应用场景有哪些？有哪些替代方案？
（1）应用场景：① 展示结构化数据（如成绩表、财务报表、商品规格表）；② 后台管理系统的数据展示（如用户列表、订单列表）；③ 需要精确对齐的内容（如日历、时间表）。
（2）替代方案：① 卡片布局（使用 CSS Grid 或 Flexbox）：适用于非严格结构化的展示（如商品列表、新闻卡片）；② 列表布局（<ul>+CSS）：适用于简单的垂直数据展示；③ 网格布局（CSS Grid）：适用于需要灵活分区的页面布局，替代传统表格布局的滥用（如页面头部、主体、底部的划分）。
